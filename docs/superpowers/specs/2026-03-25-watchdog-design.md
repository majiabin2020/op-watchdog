# 老马OpenClaw小龙虾看门狗 · 产品设计文档 (PRD)

**版本：** v1.1
**日期：** 2026-03-25
**技术栈：** Python 3.x + tkinter + pystray + PyInstaller
**交付物：** 单文件绿色exe，无需安装，无需依赖

---

## 一、产品背景与问题定义

OpenClaw 网关（`openclawgateway`）运行在 Windows 系统上，存在以下痛点：

1. 每次需要手动在 PowerShell 中输入命令启动网关
2. 误关 PowerShell 窗口会导致网关中断
3. 网关服务不稳定，会自行停止，需要手动重启
4. 重启时若上一个进程未关闭，新进程启动失败，需手动执行停止命令再重启

---

## 二、产品概述

**软件名称：** 老马OpenClaw小龙虾看门狗
**软件形态：** 无需安装的单文件绿色 exe
**核心定位：** 自动守护 OpenClaw 网关持续运行，解放用户双手

## 三、功能需求

### 3.1 主界面

- 暗黑终端极客风格（黑底，等宽字体，绿字日志）
- 窗口尺寸约 480×520px，不可调整大小
- 组成部分：
  - **标题栏**：🦞 图标 + 软件名称，右上角含最小化到托盘按钮（黄）和关闭到托盘按钮（红，非退出程序）
  - **状态区**：网关状态（在线 / 离线 / 启动中，带呼吸灯动效）、下次检查倒计时、累计重启次数
  - **日志区**：终端风格实时日志，带时间戳，颜色区分事件类型
    - 🟢 绿色：成功事件
    - 🔴 红色：异常事件
    - 🟡 黄色：操作中/进行中
    - ⚪ 灰色：常规检查
  - **按钮区**：`[ 开始看门 ]` 和 `[ 关闭看门 ]` 两个按钮
  - **底部状态栏**：版本号 + 当前状态提示
- 日志仅在程序运行期间显示，**不持久化落盘**（v1.0 范围外）

### 3.2 系统托盘

- 点击「开始看门」后主窗口隐藏，图标常驻右下角托盘
- **双击**托盘图标 → 重新显示主窗口
- **右键**托盘图标弹出菜单：
  ```
  ┌──────────────────────────┐
  │ 🦞 老马OpenClaw看门狗     │
  ├──────────────────────────┤
  │   显示主窗口              │
  │   开机自动启动  [●关]     │  ← 点击切换开/关
  ├──────────────────────────┤
  │   退出                    │
  └──────────────────────────┘
  ```
- 程序启动时读取注册表初始化"开机自动启动"的显示状态
- 主窗口右上角 ✕ → 仅隐藏到托盘，不退出程序

### 3.3 开机自启

- 默认关闭，用户通过托盘菜单手动开启/关闭
- 实现方式：写入/删除 Windows 注册表
  - 路径：`HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run`
  - 键名：`LaomaClawWatchdog`
  - 无需管理员权限

### 3.4 Windows 气泡通知

触发时机及通知内容：

| 事件 | 通知标题 | 通知内容 |
|------|----------|----------|
| 网关异常退出 | ⚠️ 网关已停止 | OpenClaw网关异常退出，正在自动重启... |
| 重启成功 | ✅ 网关已恢复 | OpenClaw网关已恢复正常运行 |
| 启动/重启失败（3次后） | ❌ 操作失败 | 网关启动失败，请手动检查OpenClaw |

### 3.5 「关闭看门」与「退出」的行为区别

| 操作 | 来源 | 行为 |
|------|------|------|
| 关闭看门 | 主窗口按钮 | 停止监控线程，主窗口显示停止状态，程序**继续运行**，托盘图标保留，可重新点击「开始看门」 |
| 退出 | 托盘菜单「退出」 | 停止监控线程，程序**完全退出**，托盘图标消失 |
| 点击窗口 ✕ | 主窗口关闭按钮 | 仅隐藏窗口到托盘，监控**继续运行** |

## 四、核心逻辑

### 4.1 网关存活检测（双重验证）

同时满足以下两个条件才判定为**在线**：
1. `tasklist` 中存在含 `openclawgateway` 的进程
2. 本地端口 `18790` 可连接（`socket.connect` 超时2秒）

任一不满足则判定为**离线/异常**。

> ⚠️ 注意：处于"启动中（STARTING）"状态时，定时监控循环跳过本轮验证，防止在网关启动窗口期内误触发异常重启。

### 4.2 网关状态机

```
状态：STOPPED（停止）/ STARTING（启动中）/ RUNNING（运行中）/ RESTARTING（重启中）

STOPPED  ──[用户点击开始看门]──→  STARTING
STARTING ──[双重验证通过]──────→  RUNNING
STARTING ──[3次全部超时/失败]──→  STOPPED（弹出失败通知）
RUNNING  ──[定时检查失败]────────→  RESTARTING（弹出异常通知）
RESTARTING ──[双重验证通过]─────→  RUNNING（弹出恢复通知）
RESTARTING ──[失败3次]───────────→  STOPPED（弹出失败通知，停止自动重试）

注：STOPPED 状态下监控线程仍保留，用户可重新点击「开始看门」触发 STARTING
```

### 4.3 启动流程（STOPPED → STARTING → RUNNING）

```
[用户点击「开始看门」，进入 STARTING 状态]
        ↓
   双重验证网关是否已在线？
     ├─ 在线 → 直接进入 RUNNING，日志显示「网关已在运行，开始监控」
     └─ 离线 → 启动流程：
         ① 检查是否有残留 openclawgateway 进程
            ├─ 有残留 → 执行 openclaw gateway stop，等待3秒
            │           仍有残留 → taskkill /F /IM openclawgateway.exe 强制清理
            └─ 无残留 → 跳过
         ② 弹出 PowerShell 窗口执行 openclawgateway 命令
            （PowerShell 窗口对用户可见，启动成功后自动关闭）
         ③ 每2秒做一次双重验证，最多等待30秒
            ├─ 验证通过 → 进入 RUNNING
            └─ 30秒超时 → 执行①清理残留，重新执行②
                          含首次共尝试3次，全部失败 → 进入 STOPPED
                          日志报错 + 气泡通知「❌ 网关启动失败，请手动检查」
```

### 4.4 定时监控循环（RUNNING 状态下）

- 监控线程在后台运行，每 **5分钟** 执行一次双重验证
  - 5分钟间隔为固定值，v1.0 不支持用户配置；可在 `config.py` 中调整常量
- 处于 STARTING / RESTARTING 状态时，跳过本轮检查
- 主界面状态区实时显示下次检查的倒计时

### 4.5 异常重启流程（RUNNING → RESTARTING → RUNNING/STOPPED）

```
[定时检查发现网关离线，进入 RESTARTING 状态]
        ↓
   弹出气泡通知：⚠️ 网关已停止，正在自动重启...
        ↓
   第1步：执行 openclaw gateway stop
   第2步：等待3秒
   第3步：检查残留进程？
     ├─ 已清理 → 进入重启
     └─ 仍残留 → taskkill /F /IM openclawgateway.exe 强制清理
   第4步：弹出 PowerShell 窗口执行 openclawgateway
          （PowerShell 窗口对用户可见，重启成功后自动关闭）
   第5步：最多等待30秒做双重验证（含首次共尝试3次）
     ├─ 成功 → 进入 RUNNING
               弹出通知：✅ 网关已恢复
               重启次数计数+1，更新日志
     └─ 3次全部失败 → 进入 STOPPED
                       弹出通知：❌ 重启失败，请手动检查
                       停止自动重试，等待用户手动点击「开始看门」重置
```

## 五、技术方案

### 5.1 技术选型

| 模块 | 技术 | 说明 |
|------|------|------|
| GUI框架 | tkinter | Python内置，无需额外依赖，黑色终端风格易实现 |
| 系统托盘 | pystray + Pillow | 托盘图标及菜单 |
| 进程管理 | subprocess + psutil | subprocess 调用命令，psutil 精确查找进程（避免通配符误杀） |
| 端口检测 | socket | 连接 localhost:18790 验证网关可用性 |
| 注册表操作 | winreg | 开机自启写入/删除 |
| 打包 | PyInstaller | `--onefile --noconsole --icon` 打包为单exe |

### 5.2 项目结构

```
op-watchdog/
├── main.py                 # 程序入口，初始化主窗口和托盘
├── config.py               # 全局常量配置（端口、超时、重试次数、检测间隔等）
├── ui/
│   └── main_window.py      # 主窗口 UI
├── core/
│   ├── watchdog.py         # 监控守护线程，状态机管理
│   └── gateway.py          # 网关管理器（启动/停止/检测）
├── utils/
│   ├── tray.py             # 系统托盘管理
│   ├── notifier.py         # 气泡通知
│   └── autostart.py        # 开机自启注册表操作
├── assets/
│   └── icon.ico            # 软件图标（🦞 小龙虾）
├── build.bat               # 一键打包脚本
└── requirements.txt        # pystray, Pillow, psutil
```

### 5.3 config.py 核心常量

```python
GATEWAY_PORT = 18790          # OpenClaw 网关默认端口
MONITOR_INTERVAL = 300        # 监控检查间隔（秒），默认5分钟
STARTUP_TIMEOUT = 30          # 单次启动等待超时（秒）
MAX_RETRY_COUNT = 3           # 最大启动/重启尝试次数（含首次）
CHECK_INTERVAL = 2            # 启动/重启等待时的检测间隔（秒）
STOP_WAIT = 3                 # 执行stop命令后等待时间（秒）
AUTOSTART_KEY = "LaomaCrawWatchdog"
```

### 5.4 打包命令

```bash
pyinstaller --onefile --noconsole --icon=assets/icon.ico \
            --name="老马OpenClaw小龙虾看门狗" main.py
```

## 七、验收标准

| # | 验收项 | 通过条件 |
|---|--------|----------|
| 1 | 无依赖运行 | 在未安装 Python 的干净 Windows 环境中双击 exe 直接启动 |
| 2 | 自动启动网关 | 点击「开始看门」后，60秒内日志显示「网关启动成功」且端口18790可连接 |
| 3 | 残留进程清理 | 手动让网关进程悬空（不关端口），点击「开始看门」，程序自动清理后成功启动 |
| 4 | 定时监控 | 程序运行期间，每5分钟日志输出一次检查记录 |
| 5 | 自动重启 | 手动kill网关进程，5分钟内日志显示「网关重启成功」且端口18790可连接，同时收到气泡通知 |
| 6 | 托盘常驻 | 点击「开始看门」后主窗口消失，托盘图标出现；双击托盘图标可重新打开主窗口 |
| 7 | 开机自启 | 托盘菜单开启「开机自动启动」后，注册表中存在对应键值；关闭后键值消失 |
| 8 | 关闭看门不退出 | 点击「关闭看门」后监控停止，但程序和托盘图标仍在，再次点击「开始看门」可恢复监控 |
